# Posts

Each published or production-ready post lives in its own folder here.

## Folder Structure

```
Posts/
└── [keyword-slug]/
    ├── article.md        ← Final publish-ready article
    └── images/
        ├── image-briefs.md   ← Image briefs from Image Brief Subagent
        └── [image assets once produced — .webp files]
```

## Naming Convention

- Folder name: `[primary-keyword-slug]` (e.g., `dominos-loyalty-program`)
- Article file: always `article.md`
- Image briefs: always `images/image-briefs.md`
- Final image assets: `[keyword-slug]-[descriptor]-[##].webp` (per CMS file name in each brief)
