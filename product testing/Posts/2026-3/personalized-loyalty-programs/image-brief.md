## Design Notes

**Overall Approach:**
- All case study images follow a **consistent card layout** (Frost cards, equal-weight benefit/earning display)
- Each image tells a mini-story: brand name → key personalization tactic → what customers earn/get
- Avoid showing actual brand logos if possible (focus on concept visualization instead)
- Use Cherokee (`#FCD299`) only for dividers/underlines, never for large text
- Never use old brand colors: `#01A6EA`, `#DC6578`, `#06048C`, `#FBC80B`

**Color Strategy:**
- **Navy** (`#06038D`) — primary text, card backgrounds
- **Cerulean** (`#00BCE1`) — badge labels, tier markers, earning activity labels
- **Frost Glass** (`rgba(255,255,255,0.18)`) — all case study cards (equal-weight benefit display)
- **White** (`#FFFFFF`) — text inside dark cards (if applicable)

**Typography:**
- Card title: **28px weight 700**
- Card description: **22px weight 600**
- Overline/category labels: **16px weight 600**, uppercase, `letter-spacing: 1.5px`
- Minimum body text weight: **600** (never use 400)

**Background:**
- **Hero:** `Raw data/Backgrounds/thumbnail/thumbnail.png` (has logo baked in)
- **Case Studies:** Use specific PNG files per image brief (e.g., content/1.png, content/2.png, etc.)
- All backgrounds have logo baked in — NO HTML overlay needed
- In HTML: Use placeholder `{{BACKGROUND_PNG}}`
- Script flag: Pass absolute path with `--background "c:/Users/AvadaGroup/Documents/Joy Content/Raw data/Backgrounds/content/[X].png"`
- Do NOT use CSS gradient — always PNG with logo baked in

**Font:**
- System sans-serif stack: `font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- No custom webfonts or Google Fonts
