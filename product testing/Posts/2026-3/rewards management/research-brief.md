## D/ Main Intent & User Context

### Search intent

The SERP mixes two audiences: HR professionals searching for employee rewards management, and ecommerce operators searching for customer loyalty program management. The dominant intent is **informational/instructional** — users want a clear definition + actionable framework.

Joy's article must **disambiguate early** (this is not an HR article) and then own the ecommerce definition completely.

### User context (ICP) — Core user problems

- Has a loyalty program set up but doesn't know if it's actually working
- Earning points accumulating but low redemption — feels like the program is inert
- No visibility into which loyalty actions are driving repeat purchases
- Wants to justify the loyalty program investment to leadership with real revenue data
- Managing rewards manually, looking for automation via triggers

### What this article MUST achieve

1. Immediately disambiguate from the HR meaning of "rewards management"
2. Define rewards management for ecommerce in a snippet-optimized 50–65 word block
3. Give a 5-component framework that maps directly to Joy's feature set

5. Deliver a Shopify setup framework the reader can execute this week
6. Introduce Joy naturally as the platform that enables all 5 components
