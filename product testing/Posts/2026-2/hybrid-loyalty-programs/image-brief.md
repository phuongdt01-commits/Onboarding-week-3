## Design Notes

**Overall Approach:**
- All case study images follow **consistent card layout** (Frost glass cards, equal-weight display)
- Each image tells a mini-story: brand → mechanics combo → what customers get
- Each case study demonstrates how hybrid mechanics *stack* to create stronger retention than single-mechanic programs
- Avoid showing actual brand logos if possible; focus on program mechanics visualization

**Color Strategy:**
- **Navy** (`#06038D`) — primary text, headings
- **Cerulean** (`#00BCE1`) — badges, tier markers, highlights, icons
- **Frost Glass** (`rgba(255,255,255,0.18)`) — all case study and framework cards
- **Cherokee** (`#FCD299`) — accent dividers/underlines only (never large text)
- **White** (`#FFFFFF`) — text inside dark cards (if applicable)

**Typography:**
- Card title: **28px weight 700**
- Card description: **22px weight 600**
- Overline/category labels: **16px weight 600**, uppercase, `letter-spacing: 1.5px`
- Minimum body text weight: **600** (never use 400)

**Background:**
- **Hero:** `Raw data/Backgrounds/thumbnail/thumbnail.png` (has logo baked in)
- **Content Images:** Rotate through content backgrounds (1–3.png per image brief recommendation)
- All backgrounds have logo baked in — NO HTML overlay needed
- In HTML: Use placeholder `{{BACKGROUND_PNG}}`
- Script flag: Pass absolute path with `--background "c:/Users/AvadaGroup/Documents/Joy Content/Raw data/Backgrounds/content/[X].png"`
- Do NOT use CSS gradient — always PNG with logo baked in

**Font:**
- System sans-serif stack: `font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- No custom webfonts or Google Fonts
